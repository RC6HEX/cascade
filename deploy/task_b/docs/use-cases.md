# Юз-кейсы

### UC-01: Создать задачу
**Источник:** БТ-01, БТ-02
**Актор:** Пользователь
**Предусловие:** Пользователь находится на главной странице приложения.
**Основной поток:**
1. Пользователь нажимает кнопку «Новая задача».
2. Система отображает форму с полями «Название» и «Описание».
3. Пользователь заполняет поля «Название» и «Описание» и нажимает «Создать».
4. Система валидирует, что поле «Название» не пустое.
5. Система создаёт задачу со статусом «К выполнению» и отображает её на доске.
**Альтернативный поток — Пустое название:**
4a. Если поле «Название» пустое, система показывает сообщение «Название задачи обязательно».
**Постусловие:** Новая задача создана со статусом «К выполнению» и отображается на доске, или задача не создана, и пользователь получил сообщение об ошибке.

### UC-02: Изменить статус задачи
**Источник:** БТ-03, БТ-02
**Актор:** Пользователь
**Предусловие:** На доске есть хотя бы одна задача.
**Основной поток:**
1. Пользователь перетаскивает карточку задачи в другую колонку (или нажимает кнопку смены статуса).
2. Система обновляет статус задачи на соответствующий выбранной колонке (или кнопке) и перемещает карточку задачи.
**Постусловие:** Статус задачи обновлен, и карточка задачи перемещена в соответствующую колонку.

### UC-03: Удалить задачу
**Источник:** БТ-04
**Актор:** Пользователь
**Предусловие:** На доске есть хотя бы одна задача.
**Основной поток:**
1. Пользователь нажимает кнопку удаления на карточке задачи.
2. Система запрашивает подтверждение удаления.
3. При подтверждении удаления, система удаляет задачу.
**Постусловие:** Задача удалена с доски.
