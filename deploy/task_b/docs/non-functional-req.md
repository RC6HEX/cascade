# Нефункциональные требования

### НФТ-01: Локализация интерфейса
**Категория:** Локализация
**Описание:** Все элементы пользовательского интерфейса, включая надписи, сообщения и подсказки, должны быть представлены на русском языке.
**Критерий приёмки:** 100% текстовых элементов, видимых пользователю, отображаются на русском языке.
**Источник:** Features

### НФТ-02: Совместимость с веб-браузерами
**Категория:** Совместимость
**Описание:** Приложение должно корректно отображаться и функционировать в распространённых современных веб-браузерах.
**Критерий приёмки:** Приложение корректно отображается и полностью функционально в последних стабильных версиях Google Chrome, Mozilla Firefox, Microsoft Edge и Safari на настольных компьютерах.
**Источник:** БТ-07

### НФТ-03: Сохранение данных задач
**Категория:** Надёжность
**Описание:** Созданные и изменённые задачи, а также их статусы, должны сохраняться в локальном хранилище браузера и быть доступны при следующем открытии приложения.
**Критерий приёмки:** После создания 5 задач, изменения их статусов и закрытия/повторного открытия вкладки браузера, все 5 задач отображаются с корректными данными и статусами.
**Источник:** БТ-06, Features

### НФТ-04: Отзывчивость интерфейса при взаимодействии
**Категория:** Производительность
**Описание:** Пользовательский интерфейс должен быстро реагировать на действия пользователя, такие как создание задачи, смена статуса (drag-and-drop) и удаление, обеспечивая плавный пользовательский опыт.
**Критерий приёмки:** Время отклика интерфейса на действия создания, перемещения или удаления задачи не превышает 500 мс при работе с 50 задачами.
**Источник:** БП (основные потоки)

### НФТ-05: Обработка ошибок ввода
**Категория:** Удобство использования
**Описание:** Система должна предоставлять чёткие и немедленные сообщения об ошибках при некорректном вводе данных, например, при попытке создать задачу без названия.
**Критерий приёмки:** При попытке создать задачу с пустым полем "Название", сообщение об ошибке "Название задачи обязательно" отображается в течение 200 мс.
**Источник:** БП (Альтернативный поток — пустое название)

### НФТ-06: Защита от XSS
**Категория:** Безопасность
**Описание:** Приложение должно быть защищено от атак межсайтового скриптинга (XSS) при отображении пользовательских данных (название, описание задачи).
**Критерий приёмки:** Ввод HTML-тегов или JavaScript-кода в поля "Название" или "Описание" не приводит к их исполнению или некорректному отображению (например, теги должны быть экранированы).
