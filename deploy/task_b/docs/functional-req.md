# Функциональные требования

### ФТ-01: Загрузка задач из локального хранилища
**Источник:** БТ-06, Features, НФТ-03
**Описание:** При запуске приложения система должна проверить наличие сохраненных задач в `localStorage` и отобразить их на доске в соответствующих колонках.
**Входные данные:** Запуск приложения (открытие страницы в браузере).
**Ожидаемый результат:** Если в `localStorage` по ключу `taskboardData` есть данные о задачах, они считываются, и задачи отображаются в колонках согласно их статусам. Если данных нет, доска отображается пустой.
**Граничные случаи:**
- `localStorage` пуст или ключ с задачами отсутствует. -> Отображается пустая доска без ошибок в консоли.
- Данные в `localStorage` имеют невалидный формат (не JSON, нарушена структура). -> Система обрабатывает ошибку (например, очищает невалидные данные) и отображает пустую доску.

### ФТ-02: Отображение Канбан-доски
**Источник:** БТ-02, БТ-07, Features
**Описание:** Система должна отображать интерфейс в виде Канбан-доски с тремя колонками, соответствующими статусам задач.
**Входные данные:** - (отображается при загрузке страницы).
**Ожидаемый результат:** На экране отображаются три вертикальные колонки с заголовками: «К выполнению», «В работе», «Выполнено».
**Граничные случаи:**
- Нет.

### ФТ-03: Создание новой задачи (успешный сценарий)
**Источник:** БТ-01, БТ-02, БТ-06, БП (Основной поток — создание), UC-01
**Описание:** Пользователь может создать новую задачу, указав её название и описание. Новая задача всегда создается в статусе «К выполнению».
**Входные данные:**
- Название задачи (непустая строка).
- Описание задачи (строка, может быть пустой).
- Действие пользователя: нажатие кнопки «Создать».
**Ожидаемый результат:**
1. В колонке «К выполнению» появляется новая карточка задачи с указанными названием и описанием.
2. Форма создания задачи очищается или скрывается.
3. Обновленный список всех задач сохраняется в `localStorage`.
**Граничные случаи:**
- Поле «Описание» оставлено пустым. -> Задача успешно создается без описания.
- Название или описание содержат спецсимволы, HTML-теги. -> Данные сохраняются и отображаются как обычный текст (теги экранируются).

### ФТ-04: Валидация при создании задачи (неуспешный сценарий)
**Источник:** БП (Альтернативный поток — пустое название), UC-01, НФТ-05
**Описание:** Система не должна позволять создавать задачу с пустым названием.
**Входные данные:**
- Название задачи (пустая строка или строка из пробелов).
- Действие пользователя: нажатие кнопки «Создать».
**Ожидаемый результат:**
1. Задача не создается.
2. Рядом с полем «Название» или под формой отображается сообщение об ошибке: «Название задачи обязательно».
3. Данные в `localStorage` не изменяются.
**Граничные случаи:**
- Пользователь ввел только пробелы в поле «Название». -> Система должна считать это поле пустым и выдать ошибку.

### ФТ-05: Изменение статуса задачи
**Источник:** БТ-03, БТ-02, БТ-06, БП (Основной поток — смена статуса), UC-02, Features
**Описание:** Пользователь может изменять статус задачи путем перетаскивания (drag-and-drop) её карточки из одной колонки в другую.
**Входные данные:**
- Действие пользователя: перетаскивание карточки задачи из колонки A в колонку B.
**Ожидаемый результат:**
1. Карточка задачи визуально перемещается из колонки A в колонку B.
2. Внутренний статус задачи обновляется в соответствии с названием колонки B («К выполнению», «В работе» или «Выполнено»).
3. Обновленный список всех задач с новыми статусами сохраняется в `localStorage`.
**Граничные случаи:**
- Пользователь перетаскивает карточку и бросает её в той же колонке. -> Статус не меняется, карточка возвращается на свое место.
- Пользователь бросает карточку не в колонку, а в пустое место между ними. -> Перемещение отменяется, карточка возвращается в исходную колонку.

### ФТ-06: Удаление задачи
**Источник:** БТ-04, БТ-06, БП (Основной поток — удаление), UC-03
**Описание:** Пользователь может безвозвратно удалить задачу с доски после подтверждения действия.
**Входные данные:**
- Действие пользователя: нажатие на кнопку «Удалить» на карточке задачи.
- Действие пользователя: подтверждение удаления в системном диалоговом окне (`confirm`).
**Ожидаемый результат:**
1. Карточка задачи исчезает с доски.
2. Обновленный список задач (без удаленной) сохраняется в `localStorage`.
**Граничные случаи:**
- Пользователь нажимает «Удалить», но в диалоговом окне выбирает «Отмена». -> Задача не удаляется, диалоговое окно закрывается.

### ФТ-07: Фильтрация задач по статусу и отображение счетчиков
**Источник:** БТ-05, Features
**Описание:** Система отображает общее количество задач в каждом статусе и позволяет фильтровать доску, показывая задачи только выбранного статуса.
**Входные данные:**
- Действие пользователя: нажатие на кнопку фильтра с названием статуса.
**Ожидаемый результат:**
1. Над доской отображаются кнопки/индикаторы для каждого статуса с указанием количества задач в нем. Например: «К выполнению (5)», «В работе (2)», «Выполнено (10)».
2. При нажатии на кнопку фильтра (например, «В работе (2)»), на доске отображается только колонка «В работе» со всеми её задачами. Остальные колонки скрываются.
3. При повторном нажатии на активный фильтр или на кнопку «Показать все», отображаются все три колонки.
**Граничные случаи:**
- В статусе 0 задач. -> Счетчик должен показывать «0».
- Пользователь пытается перетащить задачу при активном фильтре (когда видна только одна колонка). -> Drag-and-drop должен быть заблокирован, так как нет других колонок для перемещения.
