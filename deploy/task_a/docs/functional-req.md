# Функциональные требования

### ФТ-01: Сложение двух чисел
**Источник:** БТ-01, UC-01, БП (Основной поток)
**Описание:** Система должна выполнять операцию сложения двух чисел, введенных пользователем, и отображать результат.
**Входные данные:** Первое число (операнд 1), операция "+", второе число (операнд 2), команда "=".
**Ожидаемый результат:** На экране калькулятора отображается корректный результат сложения операнда 1 и операнда 2.
**Граничные случаи:**
- Сложение с нулем (A + 0 = A).
- Сложение положительного и отрицательного числа.
- Сложение двух отрицательных чисел.
- Сложение дробных чисел.

### ФТ-02: Вычитание двух чисел
**Источник:** БТ-01, UC-01, БП (Основной поток)
**Описание:** Система должна выполнять операцию вычитания одного числа из другого и отображать результат.
**Входные данные:** Первое число (операнд 1), операция "−", второе число (операнд 2), команда "=".
**Ожидаемый результат:** На экране калькулятора отображается корректный результат вычитания операнда 2 из операнда 1.
**Граничные случаи:**
- Вычитание нуля (A - 0 = A).
- Вычитание числа из самого себя (A - A = 0).
- Вычитание отрицательного числа (A - (-B) = A + B).
- Работа с дробными числами.

### ФТ-03: Умножение двух чисел
**Источник:** БТ-01, UC-01, БП (Основной поток)
**Описание:** Система должна выполнять операцию умножения двух чисел и отображать результат.
**Входные данные:** Первое число (операнд 1), операция "×", второе число (операнд 2), команда "=".
**Ожидаемый результат:** На экране калькулятора отображается корректный результат умножения операнда 1 на операнд 2.
**Граничные случаи:**
- Умножение на ноль (A * 0 = 0).
- Умножение на единицу (A * 1 = A).
- Умножение двух отрицательных чисел (результат положительный).
- Работа с дробными числами.

### ФТ-04: Деление двух чисел
**Источник:** БТ-01, UC-01, БП (Основной поток)
**Описание:** Система должна выполнять операцию деления одного числа на другое и отображать результат.
**Входные данные:** Первое число (операнд 1), операция "÷", второе число (операнд 2, не равное нулю), команда "=".
**Ожидаемый результат:** На экране калькулятора отображается корректный результат деления операнда 1 на операнд 2.
**Граничные случаи:**
- Деление нуля на число (0 / A = 0, где A ≠ 0).
- Деление числа на само себя (A / A = 1, где A ≠ 0).
- Деление, приводящее к длинной или бесконечной дроби (результат должен быть округлен до разумного количества знаков после запятой, например, 8).

### ФТ-05: Обработка деления на ноль
**Источник:** БТ-04, UC-01 (Альтернативный поток), БП (Альтернативный поток — деление на ноль)
**Описание:** Система должна предотвращать операцию деления на ноль и информировать пользователя об ошибке.
**Входные данные:** Любое число (операнд 1), операция "÷", число 0 (операнд 2), команда "=".
**Ожидаемый результат:** На экране калькулятора отображается текстовое сообщение "Ошибка: деление на ноль". Дальнейшие вычисления блокируются до нажатия кнопки сброса "C".
**Граничные случаи:**
- Попытка деления 0 на 0. Результат должен быть таким же сообщением об ошибке.

### ФТ-06: Очистка ввода и сброс состояния
**Источник:** БТ-03, UC-02, БП (Альтернативный поток — сброс)
**Описание:** Система должна предоставлять пользователю возможность очистить текущее поле ввода и сбросить внутреннее состояние калькулятора (сохраненные операнды, операция) в исходное.
**Входные данные:** Нажатие на кнопку "C".
**Ожидаемый результат:** Поле ввода очищается и отображает "0". Все сохраненные операнды и операции сбрасываются. Состояние ошибки (например, после деления на ноль) сбрасывается. Калькулятор готов к новому вычислению.
**Граничные случаи:**
- Нажатие "C" на уже сброшенном калькуляторе (состояние остается исходным, ошибок не возникает).

### ФТ-07: Добавление операции в историю вычислений
**Источник:** БТ-02, Features (История вычислений)
**Описание:** После каждого успешного вычисления (нажатия "=") система должна добавлять полную запись об операции в начало списка истории текущей сессии.
**Входные данные:** Успешно выполненная операция (операнд 1, операция, операнд 2, результат).
**Ожидаемый результат:** В список истории добавляется новая запись в формате "Операнд1 Операция Операнд2 = Результат" (например, "5 + 3 = 8").
**Граничные случаи:**
- Операции, завершившиеся ошибкой (например, деление на ноль), не должны добавляться в историю.

### ФТ-08: Отображение истории вычислений
**Источник:** БТ-02, Features (История вычислений)
**Описание:** Система должна отображать список из последних 10 вычислений. Если вычислений становится больше 10, самые старые должны удаляться из списка.
**Входные данные:** Состояние списка истории вычислений.
**Ожидаемый результат:** Справа от основного интерфейса калькулятора отображается список последних вычислений. Список содержит не более 10 записей. При добавлении 11-й записи самая первая (самая старая) удаляется.
**Граничные случаи:**
- Отображение пустого блока истории при первом запуске приложения до совершения первого вычисления.

### ФТ-09: Ввод данных с клавиатуры (цифры и операции)
**Источник:** Features (Поддержка ввода с клавиатуры)
**Описание:** Система должна позволять пользователю вводить числа и выбирать операции с помощью физической клавиатуры.
**Входные данные:** Нажатие клавиш "0"-"9", ".", "+", "-", "*", "/".
**Ожидаемый результат:** Нажатие цифровой клавиши или клавиши точки эквивалентно нажатию соответствующей кнопки на экране. Нажатие клавиши операции эквивалентно нажатию соответствующей кнопки операции.
**Граничные случаи:**
- Ввод символов, не являющихся цифрами или операциями (должны игнорироваться).
- Обработка ввода с цифрового блока (NumPad).

### ФТ-10: Управление калькулятором с клавиатуры (команды)
**Источник:** Features (Поддержка ввода с клавиатуры)
**Описание:** Система должна обрабатывать управляющие клавиши для выполнения основных действий калькулятора.
**Входные данные:** Нажатие клавиш "Enter" или "=", "Escape", "Backspace".
**Ожидаемый результат:**
- "Enter" или "=": эквивалентно нажатию кнопки "=".
- "Escape": эквивалентно нажатию кнопки "C" (полный сброс).
- "Backspace": удаляет последний введенный символ в текущем числе.
**Граничные случаи:**
- Нажатие "Backspace" на пустом поле ввода или на результате вычисления (должно игнорироваться или сбрасывать ввод к "0").

### ФТ-11: Отображение интерфейса при загрузке
**Источник:** БТ-05, UC-03, Features
**Описание:** При открытии веб-страницы система должна отобразить готовый к работе интерфейс калькулятора "QuickCalc".
**Входные данные:** Загрузка страницы приложения в браузере.
**Ожидаемый результат:** Отображается интерфейс калькулятора с названием "QuickCalc". Поле ввода содержит "0". Панель истории пуста. Все кнопки активны.
**Граничные случаи:**
- Отсутствуют.
