# Нефункциональные требования

### НФТ-01: Кроссбраузерная совместимость
**Категория:** Совместимость
**Описание:** Приложение должно корректно отображаться и функционировать во всех основных современных веб-браузерах.
**Критерий приёмки:** Приложение корректно отображается и функционирует в последних стабильных версиях Google Chrome, Mozilla Firefox, Microsoft Edge и Apple Safari на настольных компьютерах.
**Источник:** БТ-05

### НФТ-02: Скорость выполнения операций
**Категория:** Производительность
**Описание:** Выполнение базовых арифметических операций должно происходить мгновенно после нажатия кнопки "=".
**Критерий приёмки:** Время от нажатия кнопки "=" до отображения результата на экране не должно превышать 100 мс для всех базовых операций.

### НФТ-03: Отзывчивость интерфейса
**Категория:** Удобство использования
**Описание:** Пользовательский интерфейс должен быстро реагировать на действия пользователя (нажатия кнопок, ввод с клавиатуры).
**Критерий приёмки:** Задержка между действием пользователя (например, нажатием кнопки) и визуальным откликом интерфейса (например, отображением цифры в поле ввода) не должна превышать 50 мс.
**Источник:** Features (Поддержка ввода с клавиатуры)

### НФТ-04: Обработка ошибок
**Категория:** Надёжность
**Описание:** Система должна корректно обрабатывать ошибочные ситуации, такие как деление на ноль, без сбоев и с предоставлением информативного сообщения.
**Критерий приёмки:** При попытке деления на ноль приложение должно отобразить сообщение «Ошибка: деление на ноль» и оставаться в рабочем состоянии, позволяя пользователю продолжить работу.
**Источник:** БТ-04

### НФТ-05: Язык интерфейса
**Категория:** Локализация
**Описание:** Весь текстовый контент пользовательского интерфейса приложения должен быть представлен на русском языке.
**Критерий приёмки:** Все кнопки, заголовки, сообщения об ошибках и другие текстовые элементы интерфейса отображаются на русском языке.
**Источник:** Features

### НФТ-06: Защита от клиентских уязвимостей
**Категория:** Безопасность
**Описание:** Приложение должно быть защищено от распространённых клиентских уязвимостей, таких как межсайтовый скриптинг (XSS).
**Критерий приёмки:** Приложение не должно позволять выполнение произвольного кода через ввод данных в поля калькулятора или URL-параметры, а также должно корректно экранировать пользовательский ввод.

### НФТ-07: Поддерживаемость кода
**Категория:** Поддержка
**Описание:** Код приложения должен быть написан в соответствии с общепринятыми стандартами кодирования и иметь модульную структуру для облегчения дальнейшей разработки и поддержки.
**Критерий приёмки:** Код соответствует правилам линтера (например, ESLint) без критических ошибок и предупреждений, и имеет чёткое разделение логики на модули (например, UI, логика вычислений, история).
