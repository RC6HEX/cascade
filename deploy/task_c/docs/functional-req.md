# Функциональные требования

### ФТ-01: Загрузка и отображение курсов валют при открытии страницы
**Источник:** БТ-02, БТ-07, UC-01, БП (Основной поток 1-2), Features
**Описание:** При открытии страницы приложения система должна запросить актуальные курсы валют из внешнего API, закешировать их и отобразить основной интерфейс конвертера.
**Входные данные:** Пользователь открывает URL приложения в веб-браузере.
**Ожидаемый результат:**
1. Система выполняет GET-запрос к API `https://open.er-api.com/v6/latest/USD`.
2. При успешном ответе (HTTP 200) система сохраняет полученные курсы и текущую метку времени (timestamp) в `localStorage` браузера.
3. На странице отображается интерфейс конвертера: два поля для выбора валют, поле для ввода суммы, поле для отображения результата и кнопка для смены валют.
4. Поля выбора валют по умолчанию установлены (например, USD и EUR).
5. Отображается актуальный курс для валютной пары по умолчанию.
**Граничные случаи:**
- API отвечает с ошибкой (не 200 OK) — см. ФТ-05.
- Отсутствует подключение к интернету — см. ФТ-05.

### ФТ-02: Выбор исходной и целевой валюты
**Источник:** БТ-01, БТ-05, UC-02, БП (Основной поток 3-4), Features
**Описание:** Система предоставляет пользователю возможность выбрать исходную и целевую валюту из предопределенного списка.
**Входные данные:**
- Пользователь кликает на выпадающий список исходной или целевой валюты.
- Пользователь выбирает валюту из списка.
**Ожидаемый результат:**
1. Выпадающие списки содержат следующий набор валют: USD, EUR, RUB, GBP, JPY, CNY, TRY, KZT, BYN, GEL, CHF, CAD.
2. После выбора новой валюты в любом из списков, система обновляет отображаемый курс обмена и пересчитывает результат конвертации, если в поле ввода суммы есть значение.
**Граничные случаи:**
- Пользователь выбирает одинаковые исходную и целевую валюты. В этом случае курс обмена должен быть равен 1, а сконвертированная сумма должна быть равна исходной.

### ФТ-03: Конвертация суммы в реальном времени
**Источник:** БТ-03, UC-02, БП (Основной поток 5-6), Features
**Описание:** Система должна автоматически рассчитывать и отображать результат конвертации каждый раз, когда пользователь изменяет значение в поле ввода суммы.
**Входные данные:** Событие `input` в поле для ввода суммы. Введенное пользователем значение.
**Ожидаемый результат:**
1. При каждом изменении значения в поле ввода суммы, система мгновенно (согласно НФТ-01) пересчитывает результат по формуле: `Результат = Сумма * Курс_целевой_валюты_к_базовой / Курс_исходной_валюты_к_базовой`.
2. Результат отображается в соответствующем поле вывода.
**Граничные случаи:**
- Поле ввода очищено или содержит "0": результат равен "0".
- Ввод нечисловых символов (буквы, спецсимволы): система игнорирует эти символы или считает значение невалидным и отображает результат "0".
- Ввод отрицательного числа: система обрабатывает его как положительное (использует модуль числа) или блокирует ввод. Предпочтительно — блокирует или игнорирует знак "-".
- Ввод дробного числа (через точку или запятую): система должна корректно обрабатывать такие числа.

### ФТ-04: Смена направления конвертации (Swap)
**Источник:** БТ-04, UC-03, БП (Альтернативный поток — смена направления)
**Описание:** Система позволяет пользователю поменять местами исходную и целевую валюты одним нажатием кнопки.
**Входные данные:** Пользователь нажимает на кнопку с иконкой «⇄».
**Ожидаемый результат:**
1. Значения, выбранные в выпадающих списках исходной и целевой валют, меняются местами.
2. Система немедленно пересчитывает и обновляет отображаемый результат конвертации на основе нового направления.
3. Отображаемый курс обмена также обновляется в соответствии с новой парой валют.
**Граничные случаи:**
- Нажатие кнопки при пустом поле ввода суммы: валюты меняются местами, результат остается "0".

### ФТ-05: Обработка недоступности API и отсутствия сети
**Источник:** БТ-06, UC-01 (Альтернативный поток), БП (Альтернативный поток — API недоступен), Features
**Описание:** Если система не может получить актуальные курсы из API (из-за ошибки сервера или отсутствия сети), она должна использовать последние сохраненные данные из кеша и уведомить об этом пользователя.
**Входные данные:**
- API `https://open.er-api.com/v6/latest/USD` возвращает ошибку (статус не 200 OK).
- Запрос к API не может быть выполнен (например, нет интернет-соединения).
**Ожидаемый результат:**
1. Система проверяет наличие закешированных курсов в `localStorage`.
2. Если кеш существует, система использует данные из него для всех расчетов.
3. На видном месте в интерфейсе (рядом с курсом) отображается текстовая пометка: «Курс от [дата]. Актуальные данные временно недоступны», где [дата] — дата и время сохранения кеша.
**Граничные случаи:**
- API недоступен, и в `localStorage` нет закешированных данных (например, первый запуск приложения в оффлайн-режиме). Система должна отобразить сообщение об ошибке, например: "Не удалось загрузить курсы валют. Проверьте подключение к сети и обновите страницу", и заблокировать поля ввода до успешной загрузки курсов.

### ФТ-06: Управление кешем курсов валют
**Источник:** БТ-06, UC-01, Features, НФТ-07
**Описание:** Система должна кешировать курсы валют в `localStorage` и использовать их, если они не устарели, для ускорения загрузки и работы в оффлайн-режиме.
**Входные данные:** Загрузка приложения.
**Ожидаемый результат:**
1. При загрузке приложения система проверяет наличие кеша курсов в `localStorage`.
2. Система проверяет метку времени сохраненных данных.
3. Если данные существуют и с момента их сохранения прошло менее 1 часа, система использует эти данные, не отправляя запрос к API.
4. Если данные устарели (прошло более 1 часа) или отсутствуют, система выполняет запрос к API (см. ФТ-01).
5. При каждом успешном ответе от API система обновляет данные и метку времени в `localStorage`.
**Граничные случаи:**
- `localStorage` недоступен в браузере пользователя (отключен или переполнен). Система должна работать без кеширования, выполняя запрос к API при каждой загрузке страницы, но не должна вызывать ошибок. Функциональность оффлайн-режима в этом случае будет недоступна.
