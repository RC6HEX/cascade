# Нефункциональные требования

### НФТ-01: Скорость конвертации
**Категория:** Производительность
**Описание:** Система должна мгновенно отображать результат конвертации при вводе суммы пользователем.
**Критерий приёмки:** Результат конвертации должен обновляться в течение 100 мс после каждого изменения значения в поле ввода суммы.
**Источник:** БТ-03, БП-6, Features

### НФТ-02: Обработка недоступности API
**Категория:** Надёжность
**Описание:** При недоступности внешнего API для получения курсов, система должна использовать последний закешированный курс и информировать пользователя об этом.
**Критерий приёмки:** В 100% случаев недоступности API, приложение должно отображать закешированный курс с пометкой "Курс от [дата]. Актуальные данные временно недоступны" в течение 2 секунд.
**Источник:** БТ-06, БП-2a, Features

### НФТ-03: Поддержка браузеров
**Категория:** Совместимость
**Описание:** Интерфейс приложения должен корректно отображаться и функционировать в основных современных веб-браузерах.
**Критерий приёмки:** Приложение должно быть полностью функционально и корректно отображаться в последних двух стабильных версиях Chrome, Firefox, Edge и Safari на десктопных и мобильных устройствах.
**Источник:** БТ-07

### НФТ-04: Визуальная тема
**Категория:** Удобство использования
**Описание:** Приложение должно использовать тёмную тему оформления с определённым акцентным цветом.
**Критерий приёмки:** Все элементы интерфейса должны соответствовать тёмной теме, а акцентные элементы (например, кнопки, активные ссылки) должны использовать цвет #10b981.
**Источник:** Features

### НФТ-05: Язык интерфейса
**Категория:** Локализация
**Описание:** Весь текстовый контент пользовательского интерфейса должен быть представлен на русском языке.
**Критерий приёмки:** 100% видимого пользователю текста в приложении должны быть на русском языке.
**Источник:** Features

### НФТ-06: Время загрузки данных
**Категория:** Производительность
**Описание:** Система должна быстро загружать актуальные курсы валют при первом открытии страницы или при истечении срока жизни кеша.
**Критерий приёмки:** Актуальные курсы валют должны быть загружены и отображены в интерфейсе не более чем за 2 секунды с момента открытия страницы (при наличии активного интернет-соединения и доступности API).
**Источник:** БТ-02, БП-2, Features

### НФТ-07: Свежесть данных
**Категория:** Надёжность
**Описание:** Отображаемые курсы валют должны быть актуальными, согласно заданному сроку жизни кеша.
**Критерий приёмки:** Если с момента последнего успешного получения курсов из API прошло более 1 часа, система должна попытаться обновить курсы при следующем запросе или открытии страницы.
**Источник:** Features
