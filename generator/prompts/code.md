ЖЁСТКИЕ ПРАВИЛА СТЕКА (нарушать запрещено):

1. **Только vanilla web**: HTML5 + CSS3 + JavaScript ES2022. **НИКАКИХ** фреймворков (React, Vue, Svelte, jQuery), сборщиков (webpack, vite), npm-зависимостей в runtime, CDN-libs (кроме случаев когда БТ явно требует внешний API через `fetch`).
2. **Никакой сборки**: страница открывается двойным кликом по `src/index.html` или через `python -m http.server`. Импорты — нативный ESM (`<script type="module">`).
3. **Структура (минимум):**
   - `src/index.html` — точка входа, минимальная разметка с `<main id="app-root">` или подобным контейнером.
   - `src/styles.css` — все стили.
   - `src/app.js` — главный модуль, инициализация, склейка.
   - `src/<feature>.js` — отдельные модули по фичам, экспортируют функции через `export`.
4. **Каждая чистая функция, реализующая ФТ, имеет JSDoc** с тегом `@implements ФТ-NN`. Это критично для трассировки.
5. **Семантичная вёрстка**: `<main>`, `<section>`, `<article>`, `<nav>`, `<header>`, `<footer>`. `<button>` для действий, не `<div onclick>`. `aria-label` на кнопки без текста, `aria-live="polite"` на области с динамическим контентом.
6. **Адаптивность**: `viewport meta`, flexible CSS (flex/grid), мобильный first.
7. **Локализация на русский** как в Features.

ОБЯЗАТЕЛЬНЫЕ ПРАВИЛА БЕЗОПАСНОСТИ И UX:

a) **HTML `<input type="number">` НЕ принимает запятую в value.** Если выводишь число с локалью (`toLocaleString('ru-RU')`), пиши его в `<input type="text" readonly>` или в обычный `<span>`/`<output>`. Никогда не присваивай `numberInput.value = "85,42"` — браузер очистит поле.
b) **Не используй `eval`, `Function()`, `innerHTML` с пользовательским вводом.** Только `textContent` или контролируемые шаблонные литералы.
c) **Все async-операции имеют try/catch.** Любой `fetch` имеет таймаут (через AbortController) и обработку 4xx/5xx.
d) **localStorage**: оборачивай в try/catch — приватный режим может бросить QuotaExceededError.
e) **Кнопки во время async-операций**: блокируй их (`disabled`), показывай состояние загрузки.
f) **Никаких console.log в продакшен-коде** (кроме `console.error` для реальных ошибок).
g) **Все идентификаторы и data-attribute** — стабильные и описательные (`data-action`, `data-id`), чтобы тесты могли их находить.
h) **При работе с DOM — кешируй ссылки** (querySelector один раз, не на каждое событие).
i) **Event delegation** для динамических списков (например, кнопок задач). Не вешай слушатели на каждый элемент.
j) **Числа с плавающей точкой**: используй `toFixed(N)` для отображения, парсь через `Number(str)` или `parseFloat(str)`.
